    HTML中src和href的区别

    发表于 2023-03-02

    src的特性

    1. 引用外部资源

    src属性（source）是用于指定嵌入到HTML文档中的外部资源的路径，例如图像、音频、视频等。src指向的资源会被浏览器直接加载并显示或执行。常见的使用方式包括：<img src="image.png">，<script src="script.js"></script>，<audio src="audio.mp3"></audio>，<video src="video.mp4"></video>等。script元素、img元素、iframe元素、video元素。

    2. 会替换元素本身的内容

    原因就是test.js的代码嵌入到了当前script元素中，导致原本的内容被替换。

    3. 会暂停其他资源的下载

    当浏览器解析到使用src的元素时，会暂停其他资源资源的下载，直到src引用资源加载、编译、执行完毕。这也是为什么script元素推荐放在html结构的底部。

    href的特性

    1. 表示超链接

    href属性（hypertext reference）是用于指定链接的URL地址，用于在文档中创建超链接，让用户可以通过点击链接跳转到另一个页面或文档。常见的使用方式包括：<a href="http://www.example.com">Example</a>，<link href="stylesheet.css" rel="stylesheet">等。

    a标签、link标签，表示外部资源与该页面的联系。

    像CSS那样影响页面观感的可以放在html结构的头部优先加载。

    核心思想上的区别

    • src代表的是网站的一部分，没有会对网站的使用造成影响。
    • href代表网站的附属资源，没有不会对网站的核心逻辑和结构造成影响。

    为什么引用CSS使用href？

    • 正如href代表的含义一样，CSS属于网站的附属资源，不影响网站核心逻辑和结构。
